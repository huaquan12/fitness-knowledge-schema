2. 分类框架

判定标准核心测试："去掉测试" — 如果把所有平台（美团/抖音/等）从世界上删除，这条知识是否仍然成立且有经营价值？

行业本征 industry_intrinsic
描述健身/运动行业的经营本质：商业模式、运营规则、因果逻辑、战略选择。与任何平台无关。
去掉测试：✅ 通过 — 去掉平台后知识完全成立
"纯私教→高客单→低会员数量即可盈亏平衡"
平台视角 platform_perspective
描述商户如何与平台互动：佣金策略、渠道选择、线上供给策略。是平台关系的产物，不是行业本质。
去掉测试：❌ 不通过 — 去掉平台后这条知识失去意义
"美团高佣金→仅上引流品→平台定位为获客渠道"
调研产物 interviewer_artifact
商户对平台条款的主观评价，或平台产品的运营数据。完全是调研视角带来的，不是行业知识。
去掉测试：❌ 不通过 — 这条知识本身就不该存在于行业知识库中
"对7%佣金和5折MEM认为过高" / "Listing提示条0.4%点击率"

6. Schema 差距分析

共同设计的 schema 有 evidence.source（区分知识来源通道）和 causal_role（区分因果方向），但当前数据完全没有使用这些字段。这导致无法在结构层面区分行业知识和平台视角产物。

当前数据结构（缺失关键字段）
{
  "facts": ["抖音佣金1.4%..."],
  "rules": ["美团不上年卡..."],
  "explicit_exclusions": ["不接受7%佣金"],
  "causal_observations": ["美团高佣金→引流品"]
}

// 问题：
// 1. 没有 evidence.source → 无法区分知识来源
// 2. 没有 causal_role → 无法区分因果方向
// 3. 没有 knowledge_domain → 无法区分行业/平台
// 4. 扁平字符串 → 无法附加元信息
// 5. 平台知识和行业知识混在一起
升级后结构（对齐 schema 设计）

7. 升级建议

1新增 knowledge_domain 字段
在每条知识记录中新增 knowledge_domain 字段，值为 industry_intrinsic / platform_perspective / interviewer_artifact。提取时自动标注，人工复核。
"knowledge_domain": "industry_intrinsic"
2提取阶段加入"去掉测试"
提取规则中加入前置过滤：对每条候选知识执行"去掉测试"——如果去掉所有平台后该知识不成立，则标记为 platform_perspective 或 interviewer_artifact，不进入行业知识库。
3补充 evidence.source 和 causal_role
对全部 428 条数据回填 evidence.source（当前全部是 interview，未来应从 config_extract / behavior_inference 补充）和 causal_role（drives/enables/vetoes/constrains），让因果方向可机器判读。
4补充 context.store_type
为每条知识标注适用的门店类型（如"私教工作室"、"团课工作室"、"商业健身房"），使知识可条件化应用，而非一刀切。
5平台知识单独存储，不混入行业知识库
将 platform_perspective 和 interviewer_artifact 类条目分离到独立的"平台合作知识库"，与分析层的行业知识库隔离。聚类分析仅使用 industry_intrinsic 子集。
6重新聚类：仅用 310 条行业本征数据
当前聚类结果（"定价与佣金"为最强共识）是污染产物。应仅用 310 条行业本征数据重新聚类，预期会涌现出真正的行业 know-how 主题（如"教练供给瓶颈"、"单品类 vs 综合业态"、"预付制 vs 按次制"等）。

8. 重新聚类预期

剔除 118 条污染数据后，310 条行业本征知识预计会涌现以下主题（基于人工预读）：

业态分化：预付年卡制 vs 月卡制 vs 按次付费，各自的现金流模型和增长逻辑
教练供给：教练质量是服务驱动型业态的核心瓶颈，筛选/培训/留存/激励规则
品类策略：单品类深耕（普拉提/单车/蹦极）vs 综合业态，专业化壁垒 vs 规模化能力
选址与成本：核心商圈高租金倒逼高客单 vs 社区低租金低客单走量
获客方式：口碑转介绍 vs 社交媒体 vs 地推，不同业态的获客效率差异
用户粘性：社群运营/教练IP/课程趣味性对留存的影响
